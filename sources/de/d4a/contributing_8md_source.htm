2 
3 We welcome any constructive feedback.
4 
5 If you think there is anything missing or wish to contribute to our documentation feel free to [contact us](https://lightjason.github.io/contact/).
6 For some general discussion and fast answering just use our [discussion board](https://gitter.im/LightJason) so you can send us a message immediatly.

13 ## Example / Tutorial Contribution
14 
15 Currently we are working on the basic implementation of the framework, so our examples are shown only some simple test-cases. If you have some
16 ideas, test-cases or tutorials we can publish your work on the website with your name. Do the following steps to send us your proposal:
17 
18 1. fork our [example repository](https://github.com/LightJason/Examples)
19 2. create a new branch with your source code
20 3. build full working [Maven project](https://maven.apache.org/) on your code
21 4. analyse the code the code with our styleguide and bug metric
22  1. use [Checkstyle](http://checkstyle.sourceforge.net/) to check our styleguide
23  2. configuration file is found in the [AgentSpeak repository](https://github.com/LightJason/AgentSpeak/tree/master/src/analysis/checkstyle) and can be run manually
24  3. or you can put [these lines](https://github.com/LightJason/AgentSpeak/blob/master/pom.xml#L602-L623) to your ```pom.xml``` to run it automatically
25  4. configurate your compiler settings with [these lines](https://github.com/LightJason/AgentSpeak/blob/master/pom.xml#L535-L545), so that all warnings are reported
26  5. use [FingBugs](http://findbugs.sourceforge.net/) with [these lines](https://github.com/LightJason/AgentSpeak/blob/master/pom.xml#L276-L286)
27  6. and check your bugs with a report
28 5. fork your [website repository](https://github.com/LightJason/lightjason.github.io)
29 6. put your description as draft under ```content/tutorials``` in a [Markdown](https://en.wikipedia.org/wiki/Markdown) file
30 7. send us both changes as a [pull-request](https://help.github.com/articles/creating-a-pull-request/) back

33 ## Teaching Contribution
34 
35 We are very interessed in teaching aspects of your framework. So we would like to optimize our explanation on the website for understanding
36 multi-agent systems (and the programming aspects). Our main goal is a very straight understandable way of muti-agent systems and their usage.
37 If you find anything understandable on your website or put some new (didactic) elements use the following steps:
38 
39 1. fork your [website repository](https://github.com/LightJason/lightjason.github.io)
40 2. put your description as draft under ```content/tutorials``` or ```content/knowledgebase``` in a [Markdown](https://en.wikipedia.org/wiki/Markdown) file
41 3. check your changes, so it will work and fit the automatically builds
42 4. send us the changes as a [pull-request](https://help.github.com/articles/creating-a-pull-request/) back

45 ## Issue Contribution
46 
47 Our framework is still under developing and we are glad about any issue reporting, please take to heart how to report an issue
48 
49 * Describe your issue precisely as possible
50 * Put some example code to the issue e.g. a failing unit-test or an example program, so we can reproduce the problem
51 * Test your example with the current master branch / developing version to check, if the issue exits on the current developing
52 * If you can found out, why the issue occures, just link / reference in your description the code lines which are not working correctly
53 * Before you commit your issue, check out if there is not another issue which is referencend to the same / similar problem (avoid duplication), put your description
54 to the exiting disscussion
55 
56 ## Pull-Request Contribution
57 
58 Any new idea or helpful code extension or bug fixing is very helpful for us, please take heart to send the pull-request
59 
60 * Keep on track, that all unit-tests should work with your current code, if a test is not working corretly after your changes, just fix it also
61 * Follwing our automatically styleguide checking and test your code with [FingBugs](http://findbugs.sourceforge.net/) to remove any errors
62 * Before you send the pull-request keep track with the current master branch and remove any conflict
63 * Put some description lines on your pull-request, so we can understand what the problem is and what you have done to solve it
